# WSL2 ROCm 深度学习环境使用文档

本文档说明如何使用当前已经配置好的 WSL2 + ROCm + PyTorch + ONNX Runtime MIGraphX 环境，以及如何运行当前项目的自动化 benchmark。项目脚本现在支持自动设备识别，AMD ROCm 会自动选择 MIGraphXExecutionProvider，NVIDIA CUDA 机器会自动选择 CUDAExecutionProvider。

## 1. 启动环境

打开 Windows PowerShell：

```powershell
wsl -d Ubuntu
```

进入项目环境：

```bash
cd ~/torch-rocm
source .venv/bin/activate
export LD_LIBRARY_PATH=/opt/rocm-7.2.0/lib:/opt/rocm/lib:/usr/lib/wsl/lib:$LD_LIBRARY_PATH
```

建议每次使用 ONNX Runtime MIGraphX 前都设置 `LD_LIBRARY_PATH`。

## 3. PyTorch GPU 示例

```bash
python - <<'PY'
import torch

x = torch.randn((4096, 4096), device="cuda")
y = x @ x
torch.cuda.synchronize()

print(y.shape)
print(y.device)
print("ok")
PY
```

说明：PyTorch ROCm 仍然使用 `torch.cuda` API，这是 PyTorch 的统一接口名称，不代表你在使用 NVIDIA CUDA。

## 4. 导出 PyTorch 模型到 ONNX

```bash
python - <<'PY'
import torch
import torch.nn as nn

class Model(nn.Module):
    def __init__(self):
        super().__init__()
        self.fc = nn.Linear(4, 3)

    def forward(self, x):
        return self.fc(x).relu()

model = Model().eval()
x = torch.randn(2, 4)

torch.onnx.export(
    model,
    x,
    "model.onnx",
    input_names=["input"],
    output_names=["output"],
    opset_version=18,
)

print("exported model.onnx")
PY
```

## 5. 使用 MIGraphXExecutionProvider 推理

```bash
python - <<'PY'
import numpy as np
import onnxruntime as ort

sess = ort.InferenceSession(
    "model.onnx",
    providers=["MIGraphXExecutionProvider", "CPUExecutionProvider"],
)

x = np.ones((2, 4), dtype=np.float32)
y = sess.run(None, {"input": x})[0]

print("session_providers", sess.get_providers())
print("output_shape", y.shape)
print("ok")
PY
```

正常情况下应看到：

```text
session_providers ['MIGraphXExecutionProvider', 'CPUExecutionProvider']
output_shape (2, 3)
ok
```

## 7. 常见问题

### 7.1 rocm-smi 报 amdgpu not found

在 WSL 下可能看到：

```text
ERROR:root:Driver not initialized (amdgpu not found in modules)
```

这不一定影响 PyTorch 或 MIGraphX。WSL 环境优先用下面命令验证：

```bash
rocminfo
python -c "import torch; print(torch.cuda.is_available()); print(torch.cuda.get_device_name(0))"
```

### 7.2 ONNX Runtime 没有使用 MIGraphX

先确认环境变量：

```bash
export LD_LIBRARY_PATH=/opt/rocm-7.2.0/lib:/opt/rocm/lib:/usr/lib/wsl/lib:$LD_LIBRARY_PATH
```

再确认包版本：

```bash
python - <<'PY'
import onnxruntime as ort
print(ort.__version__)
print(ort.get_available_providers())
PY
```

应为：

```text
1.23.2
['MIGraphXExecutionProvider', 'CPUExecutionProvider']
```

如果你看到 `ROCMExecutionProvider` 或版本是 `1.22.2`，说明装错了 PyPI 的 `onnxruntime-rocm`，应改装 AMD ROCm 7.2 对应的 `onnxruntime_migraphx-1.23.2` wheel。

### 7.3 WSL 提示 localhost proxy 未镜像

可能出现：

```text
WSL in NAT mode does not support localhost proxies.
```

这是 WSL 的代理提示。当前环境安装和运行已经验证成功，通常不影响本地推理。如果后续下载依赖很慢，可以单独配置 WSL 代理或换源。

## 8. 推荐工作流

日常开发建议：

```bash
wsl -d Ubuntu
cd ~/torch-rocm
source .venv/bin/activate
export LD_LIBRARY_PATH=/opt/rocm-7.2.0/lib:/opt/rocm/lib:/usr/lib/wsl/lib:$LD_LIBRARY_PATH
```

训练和普通 GPU 计算优先使用 PyTorch ROCm。ONNX 推理优先使用：

```python
providers=["MIGraphXExecutionProvider", "CPUExecutionProvider"]
```

## 9. 运行自动化 benchmark

进入 benchmark 目录：

```bash
cd ~/benchmarks
source ~/torch-rocm/.venv/bin/activate
```

复制并编辑配置：

```bash
cp ~/benchmarks/scripts/benchmark_config.example.jsonc \
   ~/benchmarks/scripts/benchmark_config.local.jsonc
vim ~/benchmarks/scripts/benchmark_config.local.jsonc
```

运行完整流程：

```bash
python ~/benchmarks/scripts/run_full_benchmark.py \
  --config ~/benchmarks/scripts/benchmark_config.local.jsonc
```

每次运行会自动创建：

```text
/home/l/benchmarks/runs/<run_id>/
```

重点查看：

```text
steps_status.md
summary.md
metadata.json
baseline_results.csv
operator_results.csv
operator_pair_summary.csv
profile_summary.csv
```

重新生成已有 run 的汇总，不重跑 benchmark：

```bash
python ~/benchmarks/scripts/run_full_benchmark.py \
  --summarize-run /home/l/benchmarks/runs/<run_id>
```
