# WSL2 + ROCm + PyTorch + ONNX Runtime MIGraphX 部署教程

本文档用于在 Windows + WSL2 环境中部署 AMD ROCm 深度学习运行环境。当前已验证硬件和版本：

- GPU: AMD Radeon RX 9070 XT
- WSL: Ubuntu 24.04.4 LTS
- ROCm: 7.2
- Python: 3.12
- PyTorch: 2.9.1+rocm7.2.0
- ONNX Runtime: 1.23.2
- ONNX Runtime GPU Provider: MIGraphXExecutionProvider

说明：本教程是 AMD ROCm 环境部署教程。benchmark 项目本身已经支持自动 provider 识别；在 NVIDIA RTX 3080/CUDA 机器上部署时，请参考 `CODEX_CLI_CUDA部署引导.md`。

## 1. Windows 前置条件

在 Windows PowerShell 中确认 WSL 状态：

```powershell
wsl -l -v
```

目标状态应类似：

```text
NAME      STATE    VERSION
Ubuntu    Stopped  2
```

如果还没有 Ubuntu 24.04：

```powershell
wsl --install -d Ubuntu-24.04
```

确认显卡为 AMD Radeon RX 9070 XT：

```powershell
pnputil /enum-devices /class Display
```

Windows 端需要安装支持 ROCm on WSL 的 AMD Adrenalin 驱动。安装后进入 WSL，确认存在 `/dev/dxg`：

```bash
ls -l /dev/dxg
```

## 2. 安装 ROCm 7.2

进入 WSL Ubuntu，以 root 或 sudo 执行：

```bash
sudo apt update
cd /tmp
wget -O amdgpu-install_7.2.70200-1_all.deb \
  https://repo.radeon.com/amdgpu-install/7.2/ubuntu/noble/amdgpu-install_7.2.70200-1_all.deb
sudo dpkg -i amdgpu-install_7.2.70200-1_all.deb
```

安装 WSL + ROCm 用户态组件：

```bash
sudo amdgpu-install -y --usecase=wsl,rocm --no-dkms
```

验证 ROCm 是否识别 GPU：

```bash
rocminfo | grep -Ei "Agent|Name:|Marketing Name:|gfx|Device Type:"
```

应能看到：

```text
Name:                    gfx1201
Marketing Name:          AMD Radeon RX 9070 XT
Device Type:             GPU
```

说明：`rocm-smi` 在 WSL 下可能报 `amdgpu not found in modules`，这不代表 PyTorch 不可用。WSL 场景优先以 `rocminfo` 和 PyTorch GPU 测试为准。

## 3. 创建普通用户

如果 WSL 默认是 root，建议创建普通用户：

```bash
sudo useradd -m -s /bin/bash l
sudo usermod -aG sudo l
echo -e "[user]\ndefault=l" | sudo tee /etc/wsl.conf
echo "l ALL=(ALL) NOPASSWD:ALL" | sudo tee /etc/sudoers.d/90-l-wsl
sudo chmod 0440 /etc/sudoers.d/90-l-wsl
```

然后在 Windows PowerShell 中重启 WSL：

```powershell
wsl --shutdown
```

重新进入 Ubuntu。

## 4. 创建 Python 虚拟环境

安装 venv：

```bash
sudo apt install -y python3.12-venv python3-pip
```

创建环境：

```bash
mkdir -p ~/torch-rocm/wheels
cd ~/torch-rocm
python3 -m venv .venv
source .venv/bin/activate
pip install --upgrade pip wheel
```

## 5. 安装 PyTorch ROCm

下载 AMD ROCm 7.2 对应 wheel：

```bash
cd ~/torch-rocm/wheels
wget -nc "https://repo.radeon.com/rocm/manylinux/rocm-rel-7.2/torch-2.9.1%2Brocm7.2.0.lw.git7e1940d4-cp312-cp312-linux_x86_64.whl"
wget -nc "https://repo.radeon.com/rocm/manylinux/rocm-rel-7.2/torchvision-0.24.0%2Brocm7.2.0.gitb919bd0c-cp312-cp312-linux_x86_64.whl"
wget -nc "https://repo.radeon.com/rocm/manylinux/rocm-rel-7.2/torchaudio-2.9.0%2Brocm7.2.0.gite3c6ee2b-cp312-cp312-linux_x86_64.whl"
wget -nc "https://repo.radeon.com/rocm/manylinux/rocm-rel-7.2/triton-3.5.1%2Brocm7.2.0.gita272dfa8-cp312-cp312-linux_x86_64.whl"
```

安装：

```bash
cd ~/torch-rocm
source .venv/bin/activate
pip install \
  wheels/torch-2.9.1+rocm7.2.0.lw.git7e1940d4-cp312-cp312-linux_x86_64.whl \
  wheels/torchvision-0.24.0+rocm7.2.0.gitb919bd0c-cp312-cp312-linux_x86_64.whl \
  wheels/torchaudio-2.9.0+rocm7.2.0.gite3c6ee2b-cp312-cp312-linux_x86_64.whl \
  wheels/triton-3.5.1+rocm7.2.0.gita272dfa8-cp312-cp312-linux_x86_64.whl
```

移除 wheel 内置 HSA runtime，让 PyTorch 使用 WSL 专用runtime：

```bash
python - <<'PY'
import pathlib, site
for base in site.getsitepackages():
    lib = pathlib.Path(base) / "torch" / "lib"
    for p in lib.glob("libhsa-runtime64.so*"):
        print("removing", p)
        p.unlink()
PY
```

验证：

```bash
python - <<'PY'
import torch
print("torch", torch.__version__)
print("hip", torch.version.hip)
print("cuda_available", torch.cuda.is_available())
print("device_count", torch.cuda.device_count())
print("device_name", torch.cuda.get_device_name(0))
x = torch.randn((2048, 2048), device="cuda")
y = x @ x
torch.cuda.synchronize()
print("matmul_ok", y.shape, y.dtype, y.device)
PY
```

## 6. 安装 ONNX Runtime MIGraphX

安装基础依赖：

```bash
cd ~/torch-rocm
source .venv/bin/activate
pip install --upgrade onnx protobuf opencv-python-headless pillow tqdm psutil onnxscript
```

安装 ROCm 7.2 对应的 ONNX Runtime MIGraphX wheel：

```bash
pip uninstall -y onnxruntime-rocm onnxruntime onnxruntime-migraphx onnxruntime_migraphx
pip install --upgrade \
  "https://repo.radeon.com/rocm/manylinux/rocm-rel-7.2/onnxruntime_migraphx-1.23.2-cp312-cp312-manylinux_2_27_x86_64.manylinux_2_28_x86_64.whl"
```

配置运行库路径：

```bash
export LD_LIBRARY_PATH=/opt/rocm-7.2.0/lib:/opt/rocm/lib:/usr/lib/wsl/lib:$LD_LIBRARY_PATH
```

验证 provider：

```bash
python - <<'PY'
import onnxruntime as ort
print("ort", ort.__version__)
print("providers", ort.get_available_providers())
PY
```

应输出：

```text
ort 1.23.2
providers ['MIGraphXExecutionProvider', 'CPUExecutionProvider']
```
